# 🗺️ AI Legal Agent - 产品演进路线图 (Roadmap)

本文档基于对当前法律科技市场的调研，结合项目现有技术架构（v2.0），规划未来的功能演进路径。我们的原则是：**业务导向、按需扩展、稳定优先**。

## 🎯 Phase 1: 核心能力稳固 (Completed)
> **目标**：构建稳定、可用的多智能体协作基座。

- ✅ **多智能体架构**: 9大专业角色（法律顾问、合同审查、尽职调查等）协同工作。
- ✅ **知识检索增强 (RAG)**: 集成法规库与判例库，提供有法可依的回答。
- ✅ **基础案件管理**: 案件录入、状态追踪、文档管理。
- ✅ **实时交互**: WebSocket 流式对话，支持实时状态反馈。

## 🚀 Phase 2: 智能进化与体验升级 (Current)
> **目标**：赋予 Agent "思考"与"记忆"能力，提升用户体验。

- ✅ **情景记忆 (Episodic Memory)**: 记录历史案件处理经验，越用越聪明。
- ✅ **动态技能 (Dynamic Skills)**: 通过 Markdown 热加载新能力（如资产挖掘、税务合规）。
- ✅ **意图路由 (Intent Routing)**: 智能识别简单咨询与复杂案件，优化响应速度。
- ✅ **案件简报生成**: 一键生成专业简报，辅助律师快速交接。
- ✅ **UI/UX 重构**: 引入 Lottie 动效、实时状态感知，解决操作无反馈问题。

## 💼 Phase 3: 业务深度垂直 (Next Steps)
> **目标**：深入律师日常业务流，提供高频、刚需的效率工具。

### 3.1 律师效率工具箱 (Priority: High)
- [ ] **智能诉讼费/利息计算器**: 内置于 Chat 界面，支持 LPR、同期贷款利率自动计算。
- [ ] **诉讼时效倒推**: 输入案由和发生时间，自动计算诉讼时效截止日。
- [ ] **法院管辖权查询**: 基于被告住所地/合同履行地，推荐管辖法院。

### 3.2 深度业务场景
- [ ] **类案智能推送**: 基于 Qdrant 向量检索，主动推送与当前案件相似的法院判例。
- [ ] **自动填表**: 一键生成立案申请表、授权委托书等标准化表格。
- [ ] **证据链梳理**: 自动从上传的文档中提取证据清单，并按时间轴排列。

## 🛡️ Phase 4: 企业级安全与合规 (Planned)
> **目标**：满足律所与大型企业的合规要求。

- [ ] **私有化部署方案**: 支持 Docker/K8s 一键私有化部署，数据不出内网。
- [ ] **数据脱敏引擎**: 在发给 LLM 前自动对人名、公司名、金额进行脱敏处理。
- [ ] **审计日志**: 全程记录 AI 的推理过程与人工干预操作，可追溯。

## 📈 市场调研依据

根据 2024-2025 法律科技市场趋势分析：
1.  **"小而美"工具更受欢迎**: 律师更倾向于使用解决具体痛点（如计算、检索）的小工具，而非庞大的全能系统。因此 Phase 3 重点在于工具箱。
2.  **数据安全是红线**: 律所对客户数据极为敏感，私有化部署是获取大客户的前提。
3.  **人机协作是主流**: AI 不是替代律师，而是作为"初级律师"（Paralegal）存在。简报生成、草稿起草等辅助功能价值最高。
